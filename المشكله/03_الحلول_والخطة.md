# الحلول الممكنة وخطة الحل

## الحل المنفذ الآن

### 1. منع duplicate submission في Streamlit

تمت إضافة دالة:

`src/ui/order/streamlit_order.py::order_submission_fingerprint`

وظيفتها إنشاء بصمة ثابتة من:

- مسار ملف Excel.
- profile المستهدف.
- limit.
- execution mode.
- resume.
- match-only.
- item-workers.
- إعدادات الخصم.
- إعدادات الذكاء الاصطناعي إن وجدت.
- باقي قيم النموذج المؤثرة.

تم تجاهل كائن `upload` نفسه لأن كائنات upload ليست مناسبة كبصمة ثابتة، وتم الاعتماد على المسار النهائي للملف بدلًا منه.

قبل بدء التشغيل، يتم التحقق:

```python
if st.session_state.get("last_order_submission_fingerprint") == fingerprint:
    st.warning("This order submission was already started. Change the form to run it again.")
    return
```

بعد بدء العملية بنجاح:

```python
st.session_state["last_order_submission_fingerprint"] = fingerprint
```

الأثر:

- نفس الطلب بنفس الملف ونفس الإعدادات لن يبدأ مرة ثانية من نفس جلسة Streamlit.
- إذا أراد المستخدم تشغيلًا مختلفًا، يغير أي قيمة مؤثرة مثل limit أو profile أو range أو execution mode.

### 2. جعل log path فريدًا جدًا

قبل الإصلاح:

```python
order_output_{int(time.time())}.log
```

بعد الإصلاح:

```python
order_output_{time.time_ns()}.log
```

الأثر:

- كل تشغيل يحصل على ملف لوج مستقل حتى عند تشغيلين متقاربين جدًا.
- يسهل ربط كل log بـ run واحد.
- يقل احتمال تشخيص خاطئ بسبب log overwritten أو mixed.
