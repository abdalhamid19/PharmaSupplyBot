# ملخص مشكلة إعادة تشغيل ORDER RUN بعد انتهاء ملف كبير

## النتيجة المختصرة

المشكلة الأرجح ليست من قراءة ملف Excel نفسه ولا من تقسيم الأصناف داخليًا، بل من مسار تشغيل الطلب من واجهة Streamlit. الواجهة كانت تسمح ببدء نفس تشغيل `ORDER RUN` مرة أخرى بنفس القيم بعد انتهاء العملية، ولم تكن تحفظ بصمة آخر طلب بدأته لتمنع إعادة الإرسال غير المقصود.

تم تنفيذ إصلاحين صغيرين ومباشرين:

- منع تشغيل نفس طلب `ORDER RUN` مرة ثانية من Streamlit إذا كانت كل قيم النموذج ونفس ملف Excel مطابقة لآخر تشغيل بدأ من نفس الجلسة.
- جعل ملف لوج تشغيل Streamlit فريدًا على مستوى النانوثانية بدل الثواني حتى لا تختلط لوجات تشغيلين قريبين جدًا أو يستبدل أحدهما الآخر.

## الملف محل الشك

الملف الذي ذكرتَه موجود فعلاً في المشروع:

`data/input/order_items/00000000097.xlsx`

تم فحصه محليًا، وظهر أنه يحتوي على:

- Sheet: `Sheet1`
- عدد الصفوف الظاهر من `openpyxl`: `754`
- عدد الأعمدة: `3`
- أول صف هو الهيدر: `الكود`, `إسم الصنف`, `الكمية المطلوبة`
- أول صنف فعلي: `90236 / ZESTORETIC 20/12.5MG / qty=1`

هذا يؤكد أن الملف من ناحية البنية الأساسية قابل للقراءة بواسطة لودر Excel الحالي.

## السبب الأساسي المرجح

السبب الأساسي المرجح بنسبة عالية: إعادة إرسال نفس تشغيل Streamlit بعد انتهاء العملية.

الدليل من الكود قبل الإصلاح:

- `src/ui/order/streamlit_order.py` كان يبدأ العملية مباشرة داخل `run_order_submission` عند وصول `submitted=True`.
- بعد انتهاء العملية، `src/ui/order/streamlit_order_process.py` كان يمسح `st.session_state["order_process"]`.
- بعد مسح حالة العملية، لا توجد أي ذاكرة تقول: هذا نفس ملف Excel ونفس profile ونفس limit ونفس الخيارات التي تم تشغيلها للتو.
- لذلك أي إعادة إرسال من الواجهة، سواء بسبب ضغط متكرر أو إعادة تنفيذ Streamlit بنفس حالة الفورم، يمكن أن يبدأ Run جديد من أول ملف Excel.

## لماذا يظهر السلوك بوضوح مع الملفات الكبيرة

الملفات الكبيرة مثل `00000000097.xlsx` تجعل المشكلة أوضح للأسباب التالية:

- التشغيل يأخذ وقتًا أطول، فيزيد احتمال ضغط المستخدم على زر التشغيل أو تحديث الصفحة أو حدوث إعادة تنفيذ واجهة Streamlit.
- عند انتهاء Run طويل، إذا بدأ Run آخر بنفس القيم، سيبدو للمستخدم كأن البوت انتهى ثم بدأ الأصناف من جديد.
- مع الملفات الصغيرة قد لا تلاحظ المشكلة لأن التشغيل ينتهي سريعًا ولا تراقب بداية دورة ثانية.
