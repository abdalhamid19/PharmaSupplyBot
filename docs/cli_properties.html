ملف شرح خصائص الـ CLI في PharmaSupplyBot

هذا الملف يشرح الأوامر والخيارات الأساسية المتاحة في واجهة سطر الأوامر الحالية، استنادًا إلى تعريفات الـ parsers الموجودة داخل المشروع.

1) ما هو الـ CLI هنا؟

الـ CLI مبني باستخدام argparse ويوفر عدة أوامر فرعية (subcommands) لإدارة:

  • تسجيل الدخول/المصادقة
  • إنشاء الطلبات تلقائيًا من Excel
  • إزالة عناصر من السلة في Tawreed
  • تصدير منتجات Tawreed إلى CSV/XLSX/TXT
  • مقارنة المخزون مع منتجات Tawreed

2) الخصائص العامة المشتركة بين معظم الأوامر

الخيارات المشتركة

  • --config: مسار ملف الإعدادات config.yaml.
  • --profile: اسم البروفايل من ملف الإعدادات مثل wardany.
  • --all-profiles: تشغيل الأمر على كل البروفايلات الموجودة.

هذه الخيارات تظهر في معظم الأوامر لأنها تسمح باختيار البيئة التي سيتم العمل عليها.

3) الأوامر المتاحة

أمر auth

الغرض

تسجيل الدخول يدويًا مرة واحدة وحفظ حالة الجلسة.

الخيارات
  • --headless: تشغيل تسجيل الدخول بدون واجهة مرئية.
  • --wait-seconds: مدة انتظار Browser قبل إيقاف التشغيل، القيمة الافتراضية 600.
python run.py auth --headless

أمر order

الغرض

إنشاء أوامر تلقائيًا من ملف Excel مع دعم تشغيل Browser أو API أو Auto.

الخيارات الرئيسية
  • --excel: مسار ملف Excel الخاص بالطلبات.
  • --limit: حد لعدد العناصر المعالجة.
  • --start-item و --end-item: تحديد نطاق العناصر داخل الورقة.
  • --resume: تخطي العناصر التي سبق معالجتها.
  • --debug-browser: فتح متصفح مرئي أثناء التشغيل.
  • --match-only: تشغيل matching فقط دون إضافة العناصر للسلة.
  • --execution-mode: اختيار الوضع auto أو api أو browser.
  • --item-workers: عدد العمال المتوازيين داخل نفس البروفايل.
  • --warehouse-mode: اختيار طريقة اختيار المستودع first_available أو max_available أو max_discount.
  • --min-discount-percent: حد أدنى للخصم لاختيار المتاجر/المستودعات.
  • --prevented-items-excel: ملف XLSX يحتوي على العناصر الممنوعة من الطلب.
خيارات AI والمراجعة
  • --matching-risk-policy: safe أو aggressive.
  • --flagged-match-action: manual-review-only أو add-to-cart.
  • --provider / --model / --api-key / --review-model.
  • --ai-accept-confidence و --ai-review-threshold.
  • --no-ai-preflight.
  • --rotation-preflight-policy.
python run.py order --excel data/input/order_items/sample.xlsx --profile wardany --limit 10

أمر remove-cart

الغرض

إزالة العناصر المتطابقة من سلة Tawreed.

الخيارات
  • --excel: ملف Excel يحتوي على العناصر المراد حذفها من السلة.
  • --debug-browser: فتح نافذة متصفح مرئية.
  • --stop-flag: ملف إشارة لإيقاف العملية بين العناصر.
  • --execution-mode: نفس الخيارات auto / api / browser.
  • --item-workers: عدد العمال المتوازيين.
  • --from-manual-review: ملف مراجعة يدوي لإزالة الصفوف غير المطابقة.
  • --manual-review-scope: current-run أو saved-decisions.
  • --manual-decision: حاليًا not_matching.
python run.py remove-cart --excel data/input/remove_items/sample.xlsx --profile wardany

أمر export-products

الغرض

تصدير منتجات Tawreed إلى ملفات CSV / XLSX / TXT.

الخيارات
  • --output-dir: مجلد الإخراج، يدعم العنصر {profile}.
  • --stem: اسم الملف بدون الامتداد.
  • --page-size: حجم الصفحات عند استدعاء API.
  • --limit: حد عدد الصفوف للتصدير (0 = الكل).
  • --debug-browser: تشغيل تصدير مع متصفح مرئي.
python run.py export-products --profile wardany --limit 100

أمر match-products

الغرض

مقارنة ملف المخزون مع منتجات Tawreed وإنتاج ملف CSV مدمج.

الخيارات
  • --excel: ملف Excel/CSV للمخزون.
  • --tawreed-csv: ملف منتجات Tawreed بصيغة CSV.
  • --output: مسار ملف الإخراج.
  • --limit / --start / --end: نطاق المعالجة.
  • --resume و --trace.
  • --no-ai: تعطيل الذكاء الاصطناعي.
  • --threshold و --ai-threshold: حدود الثقة.
  • --ai-verify-policy: score / fuzzy / all-non-exact / all.
  • --ai-search-policy: safe / review-candidates / expanded / aggressive.
  • --provider / --model / --api-key / --review-model.
  • --concurrency / --ai-search-limit.
  • --no-ai-preflight و --rotation-preflight-policy.
python run.py match-products --excel data/input/inventory.xlsx --output artifacts/output/matches.csv

4) أمثلة عملية

python run.py auth --headless
python run.py order --excel data/input/order_items/orders.xlsx --profile wardany --limit 20 --execution-mode auto
python run.py export-products --profile wardany --output-dir artifacts/{profile} --limit 500
python run.py match-products --excel data/input/inventory.xlsx --output artifacts/output/results.csv --resume --trace

5) ملاحظات مهمة

  • الأوامر تستخدم عادة ملف state/config.yaml كإعداد افتراضي.
  • بعض الخيارات مثل --limit قد تكون 0 تعني “بدون حد” أو “الكل” حسب الأمر.
  • الأمر order هو الأكثر تعقيدًا لأنه يجمع بين: matching، AI، filters، resume، وexecution modes.
